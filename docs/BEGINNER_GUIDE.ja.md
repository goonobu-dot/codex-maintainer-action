# Codex Maintainer Action やさしい解説

このページは、Codex Maintainer Action を中学生でも分かるように説明するページです。

## このプロジェクトは何か

Codex Maintainer Action は、GitHub上でプロジェクトの点検を自動実行するための道具です。

中で使っているのは `codex-maintainer-kit` です。  
それをGitHub Actionsから動かせるようにしたものです。

## 何を作るのか

実行すると、次のようなファイルを作ります。

- `OSS_MAINTENANCE_AUDIT.md`: プロジェクトの点検結果
- `MAINTAINER_BRIEF.md`: メンテナー向けのまとめ
- `CODEX_TASKS.md`: Codexに頼める作業リスト
- `codex-tasks.json`: 機械が読みやすいタスク情報
- `CODEX_REVIEW.md`: 変更内容を確認するためのメモ

## 何のために役立つか

- GitHub上で定期的に点検表を作る
- OSSメンテナンスの作業候補を見つける
- チームでレビューしやすい資料を残す
- AIに任せる前に、人間が確認する材料を作る

## たとえ話

学校で、毎週月曜日に教室の点検表を作る係を想像してください。

点検表には「掃除が必要」「黒板消しがない」「机が曲がっている」などが書かれます。

でも、その係が勝手に全部を直すわけではありません。  
先生や当番が確認して、何を直すか決めます。

Codex Maintainer Action も同じです。  
GitHub上で点検表を作りますが、最後に判断するのは人間です。

## 注意点

この道具は、自動でコードをマージしません。  
自動でリリースもしません。  
Codexに勝手な書き込み権限を渡すものでもありません。

安全にAI支援を使うための、点検と資料作成の道具です。
