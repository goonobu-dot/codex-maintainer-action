# ユースケース

このページは、Codex Maintainer Action をどう実行するか選ぶための入口です。

## 1. 最初の手動実行

初めて導入するときは `workflow_dispatch` で手動実行します。

良い出力: PRコメントなし、定期実行なしで、成果物だけを安全に確認できます。

## 2. 定期メンテナンス確認

週1回または月1回の `schedule` で、メンテナーが忘れずに確認できるようにします。

良い出力: 最新の監査、説明資料、タスク一覧、レビュー資料がartifactとして残ります。

## 3. PRごとの成果物生成

`pull_request` で動かすと、PRごとにレビュー用の材料を作れます。

良い出力: workflow run上のartifactと、必要に応じた短いPRコメント。

## 4. PRコメントモード

`comment-pr: "true"` は、workflowに `pull-requests: write` を付ける場合だけ使います。

良い出力: 長いレポートを貼り付けず、生成成果物への短い案内だけをコメントします。

## 5. CLI版との使い分け

細かく手元で確認したいときは `codex-maintainer-kit` を使います。GitHub側で繰り返し実行したいときは、このアクションを使います。
